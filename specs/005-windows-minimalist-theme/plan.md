# Implementation Plan: Windows Minimalist Keycloak Theme

**Branch**: `005-windows-minimalist-theme` | **Date**: 2026-05-03 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/005-windows-minimalist-theme/spec.md`

## Summary

Create a self-contained Keycloak login theme named `windows` under `themes/windows/login/`, inheriting from `keycloak.v2`, that applies a Windows 11 / Fluent Design visual style: soft neutral page background, frosted-glass card surface (`backdrop-filter` + semi-transparent white), Segoe UI typography, Windows accent blue (`#0078D4`) primary button, and clean minimal input fields. All PatternFly v5 DOM selectors (`pf-v5-c-*`) are targeted correctly. WCAG AA contrast is met for all text surfaces. The Dockerfile is updated to package the new theme alongside the existing `futuristic` theme.

## Technical Context

**Language/Version**: CSS 3 (custom properties, backdrop-filter, clamp)  
**Primary Dependencies**: Keycloak 26.6.1 · PatternFly v5 DOM (`keycloak.v2` base theme) · podman-compose  
**Storage**: N/A — CSS + theme.properties files only  
**Testing**: `./scripts/validate-theme-catalog.sh` (image build + theme catalog check) · visual smoke test via browser  
**Target Platform**: Keycloak 26.6.1 container on Linux (podman), rendered in a modern browser  
**Project Type**: Keycloak login theme (CSS-only customisation)  
**Performance Goals**: No measurable runtime overhead — CSS delivery only; no web font network requests  
**Constraints**: No JavaScript · no custom FreeMarker templates · no additional runtime services · Segoe UI served from OS (no web font download) · light mode only (v1)  
**Scale/Scope**: Single login CSS file + `theme.properties` + one Dockerfile line addition
